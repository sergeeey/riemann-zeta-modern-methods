# A-04 · Харди Z(t) + метод Тьюринга — sandbox черновик
Дата: 2026-06-14

## Гипотеза (сырая)
Все нули ζ до высоты T лежат на критической прямой — проверяемо НЕЗАВИСИМЫМ счётом:
число смен знака Z(t) (нули на Re=½) должно совпасть с N(T)=θ(T)/π+1 (все нули в полосе).

## Zero-Signal Gate
- **Entity:** нетривиальные нули ζ(s) в полосе 0<Im≤T
- **Falsifiable predicate:** N_found(на Re=½) = N(T)(вся полоса) ⟺ нет off-line нулей до T
- **Measurable outcome:** |N_found − θ(T)/π − 1| < 1 (PASS) vs систематический отрицательный дрейф S(T) (FAIL = off-line/пропуск)

## Почему НЕтавтологично
`find_zeros` ищет ТОЛЬКО на Re=½ (смены знака Z). N(T)=θ/π+1 считает нули в полосе,
НЕ предполагая Re. Off-line ноль → Z его не поймает, θ/π+1 посчитает → mismatch.
Тест МОЖЕТ провалиться (в отличие от Marchenko-тавтологии).

- N_found (смены знака Z, Re=½) = **138** == N_strict (`mp.nzeros`, Turing, полоса) = **138** → совпали
- **off-line нулей до T=300 НЕТ** (независимый Turing-count = счёт на прямой → строгий вывод)
- positive control: первые 10 vs Odlyzko, max err 9.4e-19; residual ≤ 1e-28; зазор-запас ×12
- **skeptic: NEEDS-STRICTER** (точечный S(T)=+0.29 ≠ интеграл Тьюринга; off-line+пропуск дали бы похожий S)
  → усилено: θ/π+1 заменён на `mp.nzeros` → **SOLID-IN-WINDOW**
- CAVEAT: `mp.nzeros` тоже mpmath (иной алгоритм, но не cross-library). Полная независимость =
